GEAAR · Flight Path Authoring

UI Redesign Explorations

Eight directions for the drone flight-path authoring tool — fresh layouts, control schemes and interaction models to mine for ideas before the Unity build. 08 is a synthesis of the elements you liked across 01, 02 and 05. Every concept depicts the same scene & feature set (paths → waypoints with speed 1–5, height/AGL, look-at targets, camera preview, timeline) so they're directly comparable. All keep a dark canvas; the path line & controls are designed per concept.

Makes time the primary axis: the bottom half becomes a multi-track editor with separate Speed, Height and Look-at lanes (segments, curves, keyframes). The 3D view shrinks to a reference at the top.

How to read these: each file is a self-contained static mockup (HTML/CSS, no logic) — open any of them directly in a browser. They're idea-generators, not working tools: study the layout, control groupings and metaphors, then cherry-pick whatever earns its place in the Unity build. Mix freely — e.g. the Editor-Dock hierarchy with the Mission-Control flight strip, or the Editorial inspector with the GIS layers panel.
